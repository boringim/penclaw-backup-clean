- 将 `openclaw.json` 中的 `gateway.auth.token` 与 `gateway.remote.token` **对齐到当前进程实际使用的 token**：`openclaw-manager-local-token`
- 同步更新 `C:\Users\Administrator\.openclaw\gateway\token.txt`
- 修复后 `sessions_list` 恢复可用

### 📌 经验教训（重要）
1. **不要粗暴删除 token 环境变量**，先判断来源：进程级 / 用户级 / 系统级
2. 如果变量是进程级注入，优先采用 **配置对齐**，不要先删，避免误伤现有依赖
3. 当出现 `gateway token mismatch` 时，先检查：
   - `$env:OPENCLAW_GATEWAY_TOKEN`
   - `openclaw.json` 里的 `gateway.auth.token`
   - `openclaw.json` 里的 `gateway.remote.token`
   - `C:\Users\Administrator\.openclaw\gateway\token.txt`
4. `openclaw status` 和工具接口可能使用不同 token 来源，看到 CLI 正常不代表 tools 一定正常，最好再测一次 `sessions_list`
